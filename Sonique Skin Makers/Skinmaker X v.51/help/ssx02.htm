SONIQUE SKINMAKER X HELP SYSTEM - INTRODUCTION TO THE PROCESS OF MAKING A SKIN


INTRODUCTION

This document is somewhat based on the "How to Make Skins" file which is included in the standard Skinpack. This version is re-arranged and has images describing some of the steps.

First of all, probably the best way of learning is to look at what others have done. SSX has that ability and you can easily load Sonique skins. If you want to test the program, at least one skin is provided in your [sonique path]\skins\examples directory.

As you must have figured out, most of the work done when creating a skin is done with image editing applications such as Adobe Photoshop (Which we strongly recommend). SSX is merely a tool to transform your images into a Sonique skin.

So what do I do?
If you are to create a full-fledged 1.x Sonique skin you need to design the following:

- The Nav Console Mode with all buttons & knobs

  • play/stop
  • pause
  • previous/next
  • load
  • shuffle
  • repeat
  • volume
  • PgUp/PgDn


- The Equalization/Audio control along with the sliders and three knobs

  • equalizer band sliders
  • amplification knob
  • balance knob
  • pitch knob


- The Middle State Mode with all buttons, song/playlist position rings and knobs
  • play/stop
  • pause
  • previous/next
  • load
  • shuffle
  • repeat
  • volume
  • Jog Wheel


- The Small State Mode (not really any user defined buttons here)



In all modes, you can choose to not include certain parts and that's up to you. There are also more areas in development - one of the new ones is the Link area on both the MSM and NAV mode.

THE SKIN FILE FORMAT

The format of the Sonique skin file (*.sgf = Sonique Graphics File) is that of an archive - pretty much like a zip file. A SGF file is also like a directory. The files are arranged inside the file like files in a normal directory. (Another name of this format is DAF = Data Archive Format)
The following files are stored in a skin-file:

  • The skin.ini - a file that contains values like information about the author, positions of buttons within the images (in addition to regions, see below).
  • JPEG images for each of the states plus three knob animations and one misc image where lit-states of buttons and the EQ drawer is stored.
  • Several RGN files (regions)





    REGIONS

    As you can see when running Sonique with different skins it's possible to do almost any design, at least in the MSM. This is achieved using regions.

    A region is an area pretty much like a drawn image in Adobe Photoshop and can also be referred to as an array of pixels. It consists of X number of 1x1 pixel squares which together builds a "region". Sonique uses these regions to define where buttons are positioned and the shape of the windows in the different modes.

    1. 2. 3.

    This is the standard way of creating a region. 1# is a blowup of an image. In 2# the red pixels together form a region that is overlayed on top of the image. In 3# the result is shown (ie. the part of the original image that was covered by the region).

    In SSX (and the old skinmaker) the regions are created by providing a 16 color BMP bitmap and then searching for pixels with a specific color that either A) are contiguous (floodfill) or B) of the same color value but possibly broken up into different places (colorkey).     THE SKIN.INI

    As a general rule, while using SSX, see the INI file comments for details. However dealing with it is still quite difficult if you aren't used to it. Especially when it comes to the misc.jpg. The standard misc.jpg looks like this:




    Once you have all of the graphics made, the most pain in the ass thing is tweaking the misc.jpg and mask locations so that the lit buttons line up with the original jpg. The misc.jpg contains all of the lit states of the buttons (what the buttons look like when they are pressed). Look at the example and you will understand. There's a rundown of everything in the misc_default.jpg and their default xy coordinates in the misc.txt file that came right out of the Sonique source code. All of the numbers here correspond with the misc_default.jpg file. Most of the locations are pretty self explanatory, but there are a few that are fuzzy:
  • redon : the nav button lit state for mid state mode
  • orangeon: The open button lit state
  • greenon: the shuffle button lit state
  • cyanon: the loop button lit state for loop all
  • greenon2: the loop button lit state for loop one
  • cyanoff: the loop button transition color (this is a button state that is displayed between loop all and loop one as an animation transition. So if your loop all button was red and your loop one button was blue, you would make this a purple color, or something that looks good).

    If you plan on changing the locations of any of the images in the misc.jpg then you have to put that in to the skin.ini file. You would do that by entering in under the [misc locations] heading:

    LOCATION_NAME_x=x_coordinate_value
    LOCATION_NAME_y=y_coordinate_value

    The [misc values] are handled pretty much the same way as the locations except for the fact that not all of them are coordinate values. The data is entered in the same way if it is changed, only it is under the [misc values] area instead (of course).

    Animations:

    The animations are really simple. Just check the examples in the jpg directory (volknob, shuttle). You set the size of the knob and the number of frames in the [misc values]. If you change it, just make the smallknob.jpg a grey color because the color is set in the [sonique colors] area.
